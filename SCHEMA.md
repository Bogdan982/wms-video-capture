# WMS Video Capture — схема работы и UI

## 🔄 Жизненный цикл задачи

```
WMS (сервер) → Intent (task_id)
        ↓
   ┌──────────┐
   │  SPLASH  │  3 сек — заставка с названием организации
   └────┬─────┘
        ↓
   ┌──────────┐
   │ CONFIRM  │  «Получен ID заказа. Начать съёмку?»
   │          │  ⚙ Настройки | QR-сканер (ручной ввод ID)
   └────┬─────┘
        ↓  удержание 2 сек
   ┌──────────┐
   │  RECORD  │  Системная камера — превью + запись
   │          │  Кнопка «Завершить запись»
   └────┬─────┘
        ↓  видео сохранено
   ┌──────────┐
   │   DONE   │  Диалог: «Использовать ID» или «Заменить через QR»
   └────┬─────┘
        ├→ [Использовать] → выгрузка
        └→ [QR-сканер] → новый ID → выгрузка
                ↓
   ┌──────────┐
   │ UPLOAD   │  Прогресс: SMB-сервер + флаг WMS
   └────┬─────┘
        ↓
   ┌──────────┐
   │  RESULT  │  ✅ Завершено / ❌ Ошибка (с повтором)
   └──────────┘
```

## 🖥️ Экраны приложения

### 1. SplashScreen (заставка)
- Фон: тёмно-синий
- Текст: «ПРОГРАММА АВТОМАТИЗАЦИИ ВИДЕОФИКСАЦИИ СКЛАДСКОГО ПРОИЗВОДСТВА»
- Название организации (из настроек)
- «Roman design © 2025»
- Автопереход через 3 сек → ConfirmScreen

### 2. ConfirmScreen (подтверждение)
- **Строка состояния**: «Сеть: OK / SMB OK / WMS OK» (зелёная/красная)
- Кнопка ⚙ (настройки) и ✕ (закрыть)
- **ID заказа**: крупно «ABC-123»
- **Кнопка «УДЕРЖИВАЙТЕ 2 СЕК»** — защита от случайного нажатия
- Прогресс-бар удержания
- **QR-сканер** (если нет Intent) — ручной ввод ID
- **⚙ Настройки** — по паролю (по умолчанию: `0000`)

### 3. RecordScreen (съёмка)
- **Превью**: Kivy Camera widget (низкое разрешение)
- **Запись**: системная камера (высокое качество)
- Таймер записи
- Кнопка «Завершить запись» (красная)

### 4. DoneScreen (завершение)
- Иконка статуса: 🎥 / ✅ / ❌
- Информация о видео
- Кнопки:
  - ✅ «Использовать ID» — выгрузка
  - 📷 «Сканировать QR-код» — замена ID
  - ✕ «Закрыть»

### 5. UploadScreen (выгрузка)
- Прогресс-бар: «Выгрузка на сервер... 45%»
- Статус: SMB + WMS флаг
- Кнопка «Повторить» (при ошибке)

### 6. ScanScreen (QR-сканер)
- Статус: «Ожидание QR-кода...»
- Результат: «ID: ABC-123»
- Кнопки: «Назад» | «Сканировать снова»

### 7. SettingsScreen (настройки)
- Пароль администратора (по умолчанию `0000`)
- Поля:
  - SMB сервер (IP)
  - SMB шара (имя)
  - SMB пользователь
  - SMB пароль
  - WMS URL
  - Логи SMB сервер
  - Логи SMB шара
- Смена пароля (минимум 4 символа)
- Сохранение автоматическое

## 🏗️ Архитектура

```
main.py                   # Точка входа, Intent, разрешения
app/
  app.py                  # WmsVideoApp (оркестратор)
  config.py               # AppConfig (JSON, SMB/WMS/пароль)
  screens/
    splash_screen.py      # Заставка
    confirm_screen.py     # Подтверждение задачи
    record_screen.py      # Запись видео
    done_screen.py        # Завершение / диалог ID
    upload_screen.py      # Выгрузка на SMB
    scan_screen.py        # QR-сканер
    settings_screen.py    # Настройки (по паролю)
  services/
    camera_service.py     # Камера (Intent + preview)
    network_service.py    # SMB/CIFS клиент (pysmb)
    wms_service.py        # WMS REST клиент
    file_service.py       # Файлы (сохранение/удаление)
    logger_service.py     # Логирование (локально + SMB)
    qr_service.py         # QR-сканер (ZXing)
    upload_service.py     # Foreground service
assets/
  config.default.json     # Настройки по умолчанию
  AndroidManifest.xml     # Кастомный (Intent filter + FileProvider)
  file_paths.xml          # Пути для FileProvider
res/xml/
  file_paths.xml          # Пути для FileProvider
```
